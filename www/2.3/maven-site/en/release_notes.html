2.3

Downloads

All distribution files for this release can be found here.

Features and changes

Some of the most noteworthy new features and changes are:

  • genesis and the samples work on JBoss 4.x, without any additional configuration
  • Some Ant tasks were replaced by others that are more efficient, resulting in a faster build
  • New documentation explain how to build WEB projects
  • A new modular, extensible and more efficient build process has been developed
  • An empty-web-project sample was created
  • All required XDoclet jars are now part of genesis distribution
  • Added two new validators: min e max
  • SessionFactory is instantiated on startup for local mode execution
  • It is possible to customize how components are created by a @DataProvider
  • A new API that makes the way values are cloned by genesis configurable has been defined

The following bugs have been fixed on this release:

  • useradmin:install does not work using a "clean" genesis installation (Issue # 263)
  • CriteriaResolverImpl is broken (Issue # 281)
  • CriteriaPropertyHelper throws a NullPointerException if the form has a @NotBound property (Issue # 252)

The full change log between 2.2 and 2.3 can be found here.

2.2

Downloads

All distribution files for this release can be found here.

Features and changes

Some of the most noteworthy new features and changes are:

  • The XDoclet extensions had their structure reworked and are now properly documented
  • Support for virtual properties has been added
  • An implementation of paging for lists is now part of the framework
  • net.java.dev.genesis.resolvers.EmptyResolverRegistry has been modified to allow net.java.dev.genesis.resolvers.EmptyResolver instances to be registered programmatically
  • The onClose() method from net.java.dev.genesis.ui.thinlet.BaseDialogThinlet has been modified in order to improve memory release (for more information, check issues 243 and 244 )
  • textareas are now made not editable instead of being disabled so it is possible to scroll them (for more information, check issue 248 )

The following bugs have been fixed on this release:

  • SessionContext.setRollbackOnly() should only be called inside a transaction (Issue # 213)
  • The DefaultConverter(boolean) constructor is broken (Issue # 218)
  • @VisibleWhen/@EnabledWhen do not work for read-only properties (Issue # 225)
  • WorkerThread's lifecycle should be bound to TimeoutAspect's one (Issue # 245)
  • Selection fails for comboboxes and lists populated using @DataProvider and containing different values for the key and value properties (Issue # 246)

The full change log between 2.1 and 2.2 can be found here.

The full documentation for this release can be found here.

2.1

Downloads

All distribution files for this release can be found here.

Features and changes

Some of the most noteworthy new features and changes are:

  • POJOs can be used to handle all types of events
  • BeanUtils converters now use the same hierarchical search scheme other genesis components use
  • Hibernate 2.1.8 now comes bundled with genesis
  • Direct references to Commons JXPath classes have been deprecated. The next major genesis release will contain generic support to script languages
  • A FAQ section has been added to the docs

The following bugs have been fixed on this release:

  • An exception is thrown when an item of a list bound to a Collection or array is selected (Issue # 192)
  • Resetting form state can throw a NullPointerException under some circumstances (Issue # 193)
  • An exception is thrown if the objectField property of a @DataProvider is an array of any Object subclass (Issue # 194)
  • The default converter registered for Object is wrong (Issue # 202)

The full change log between 0.2 and 2.1 can be found here.

Features and changes

Many changes and features are part of this release. Some of the most noteworthy are:

  • Remote and transactional methods are now defined using annotations, replacing the old marker interfaces + method naming convention
  • @Criteria implementation
  • Local execution mode
  • Brand new redesigned UI model, much more expressive and powerful
