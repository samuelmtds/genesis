3.2-RC1

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • Adicionado um validador de URL.
  • As mensagens de erro foram internacionalizadas e localizadas para o português do Brasil (pt-BR).

Alguns dos bugs corrigidos nessa versão foram:

  • Uma IllegalArgumentException é lançada quando um JComboBox/Combo é utilizado com indexField primitivo (Issue # 503).
  • Uma NestedNullException é lançada quando algum elemento de uma propriedade aninhada é nulo (Issue # 511 e # 512).
  • Os jars da aplicação são regerados desnecessariamente utilizando o Ant 1.7.1 (Issue # 498).
  • A renderização da linha em branco no JComboBox está errada (Issue # 504).

Uma lista completa das alterações ocorridas entre a 3.1 e a 3.2-RC1 pode ser obtida aqui.

A documentação completa para esta versão pode ser encontrada aqui.

3.1

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • Adicionado suporte à formatação utilizando expressões de script.
  • Não é mais necessário, na maioria dos casos, definir propriedades para utilizar @DataProvider.
  • Suporte ao JBoss 4.2.
  • Suporte a project.properties, um arquivo com as propriedades compartilhadas do projeto.

Alguns dos bugs corrigidos nessa versão foram:

  • @DataProvider não funciona para JTables que utilizam o recurso de ordenação do Java 6 (Issue # 448).
  • Uma ConcurrentModificationException é lançada se mais de um componente utiliza a estratégia "as you type" (Issue # 457).
  • @DataProvider não funciona com listas de seleção múltipla (Issue # 466).
  • @EnabledWhen/@VisibleWhen podem não funcionar para os bindings Swing/SWT (Issue # 486).

Uma lista completa das alterações ocorridas entre a 3.0 e a 3.1 pode ser obtida aqui.

A documentação completa para esta versão pode ser encontrada aqui.

3.0

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • Adicionado suporte a Swing e SWT.
  • Foi adicionado suporte a estratégias de binding.
  • O binding de botões de rádio e caixas de seleção foi revisto e documentado
  • Suporte a enums e anotações do Java 5.
  • É possível utilizar o binding em projetos sem o uso de AOP, apenas acrescentando os jars corretos ao classpath.
  • Foram melhoradas as mensagens de erro para anotações incorretas, propriedades inexistentes e arquivos não disponíveis.
  • Suporte genérico a linguagens de script: expandindo o suporte JXPath, o framework suporta diretamente BeanShell, JavaScript e EL, e ainda provê um mecanismo para suporte de outras linguagens
  • A linguagem padrão de scripts agora é Javascript.
  • Upgrade para o AspectWerkz 2.0.
  • Revisão completa da documentação.

Os seguintes bugs foram corrigidos nessa versão:

  • @BeforeAction não funciona para @Action que também seja um @DataProvider (Issue # 331).
  • A invocação de métodos de commands a partir de outros commands não funciona corretamente no modo local desktop (Issue # 446).
  • O binding de caixas de seleção e botões de rádio não funciona para Thinlet (Issue # 425).

Para obter a lista detalhada de mudanças nas versões intermediárias do 3.0 (EAs e RCs), clique aqui.

A documentação completa para esta versão pode ser encontrada aqui.

2.3

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • O genesis e os exemplos funcionam no JBoss 4.x, sem necessitar qualquer configuração
  • Algumas tarefas do Ant foram trocadas por outras mais eficientes, resultando num build ainda mais otimizado
  • Nova documentação de como criar um projeto WEB
  • Foi criado um novo processo de compilação com um build.xml modular e extensível, mais rápido e inteligente
  • Um novo esqueleto de um projeto web foi criado e disponibilizado junto com os exemplos com o nome de empty-web-project
  • Todos as bibliotecas necessárias do XDoclet estão disponíveis na distribuição do genesis, dispensando o download do XDoclet
  • Adicionado novos validadores: min e max
  • SessionFactory do hibernate é instanciado na inicialização no modo de execução local
  • É possível customizar como os componentes são criados por um @DataProvider
  • Foi definida uma API que torna configurável a clonagem de valores pelo genesis

Os seguintes bugs foram corrigidos nessa versão:

  • O exemplo useradmin não funciona com uma instalação limpa do genesis (Issue # 263)
  • CriteriaResolverImpl não funciona corretamente (Issue # 281)
  • CriteriaPropertyHelper lança uma NullPointerException se o form contiver uma propriedade @NotBound (Issue # 252)

Uma lista completa das alterações ocorridas entre a 2.2 e a 2.3 pode ser obtida aqui.

A documentação completa para esta versão pode ser encontrada aqui.

2.2

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • As extensões do XDoclet tiveram sua estrutura modificada e agora estão documentadas
  • Foi adicionado suporte a propriedades virtuais
  • Uma implementação de paginação para listas foi incluída
  • net.java.dev.genesis.resolvers.EmptyResolverRegistry foi modificada para permitir o registro programático de instâncias de net.java.dev.genesis.resolvers.EmptyResolver
  • O método onClose() de net.java.dev.genesis.ui.thinlet.BaseDialogThinlet foi modificado a fim de melhorar a liberação de memória (mais informações nos issues 243 e 244 )
  • textareas se tornam não-editáveis ao invés de serem desabilitadas de modo a ser possível utilizar a barra de rolagem (mais informações no issue 248 )

Os seguintes bugs foram corrigidos nessa versão:

  • SessionContext.setRollbackOnly() deveria ser chamado somente dentro de uma transação (Issue # 213)
  • O construtor DefaultConverter(boolean) não funciona (Issue # 218)
  • @VisibleWhen/@EnabledWhen não funcionam para propriedades somente leitura (Issue # 225)
  • O ciclo de vida de WorkerThread deveria estar ligado ao ciclo de vida de TimeoutAspect (Issue # 245)
  • A seleção falha para combos e listas populados com @DataProvider e contendo valores diferentes para key e value (Issue # 246)

2.1

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • POJOs podem ser usados para tratar todos os tipos de eventos
  • Os conversores do BeanUtils usam a mesma busca hierárquica dos outros componentes do genesis
  • A versão do Hibernate que acompanha o genesis agora é a 2.1.8
  • As referências diretas a classes do Commons JXPath foram deprecated. A próxima versão do genesis conterá suporte genérico a linguagens de script
  • Uma seção de FAQ foi adicionada à documentação

Os seguintes bugs foram corrigidos nessa versão:

  • Uma exceção é lançada quando um item numa list ligada a uma Collection ou array é selecionado (Issue # 192)
  • O reset do estado do form pode lançar uma NullPointerException sob condições específicas (Issue # 193)
  • Uma exceção é lançada se o objectField de um @DataProvider for uma array de qualquer subclasse de Object (Issue # 194)
  • O conversor padrão registrado para Object está errado (Issue # 202)

Uma lista completa das alterações ocorridas entre a 0.2 e a 2.1 pode ser obtida aqui.

A documentação completa para esta versão pode ser encontrada aqui.
