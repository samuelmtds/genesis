Usando anotações

Grande parte das funcionalidades do genesis é acessível através do uso de anotações ou metadados. Estas anotações são parte integrante da linguagem Java a partir do Java 5, mas o genesis suporta seu uso também com Java 1.4, com uma sintaxe bastante similar.

Por exemplo, considere o exemplo abaixo de uso da mesma anotação com Java 5 e Java 1.4:

import net.java.dev.genesis.annotation.Form;

@Form
public class ExemploAnotacaoClasseJava5 {
   // ...
}

/**
 * @Form
 */
public class ExemploAnotacaoClasseJava14 {
   // ...
}

Ambas as declarações da anotação @Form terão o mesmo efeito nos dois ambientes. Note que no segundo caso, conhecido como anotações Javadoc, não é necessário utilizar a instrução import para o uso da anotação.

No caso de anotações Javadoc, é permitido utilizar uma sintaxe mais livre para anotações que recebam parâmetros, conforme mostrado abaixo:

import net.java.dev.genesis.annotation.EnabledWhen;
import net.java.dev.genesis.annotation.Form;

@Form
public class ExemploAnotacaoMetodoJava5 {
   @EnabledWhen("condicao")
   public String getPropriedade() {
      // ...
   }

   // ...
}

/**
 * @Form
 */
public class ExemploAnotacaoClasseJava14 {
   /**
    * @EnabledWhen condicao
    */
   public String getPropriedade() {
      // ...
   }

   // ...
}

Não é necessária nenhuma configuração para usar a sintaxe Javadoc com Java 1.4. É possível, mas não recomendado, usar a sintaxe Javadoc mesmo com Java 5. Isso pode ser necessário devido ao uso de outras ferramentas de análise ou processamento de código que não suportem anotações Java 5. Para fazer isto, basta descomentar o seguinte trecho no arquivo build.xml do projeto vazio: 


      <property name="force.annotation" value="true" />

Nesta documentação, os exemplos usarão a sintaxe do Java 5 por padrão e serão explicadas quaisquer peculariedades que existam no uso das anotações Javadoc.
