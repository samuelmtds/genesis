0.2-RC2

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Esta versão contém somente poucas melhorias em relação a 0.2-RC1:

  • É possível colocar mais de uma anotação @AfterAction e/ou @BeforeAction no mesmo método
  • O exemplo useradmin foi atualizado para utilizar novas funcionalidades do genesis 0.2-RC1
  • O build.xml do empty-project suporta novas propriedades para definição dos parâmetros de acesso ao banco de dados

Os seguintes bug foram corrigidos nessa versão:

  • A propriedade objectField é setada para null ao invés de uma coleção vazia quando a propriedade é uma Collection e o @DataProvider está ligado a uma tabela (Issue # 173)
  • A propriedade indexField é ignorada inicialmente ao fazer o binding e caso um objectField que seja uma array ou Collection seja usada em conjunto com ele, o uso do valor -1 lança uma ArrayIndexOutOfBoundsException (Issue # 181)

A lista completa das alterações ocorridas entre o 0.2-RC1 e 0.2-RC2 pode ser obtida aqui.

A documentação completa para esta versão pode ser encontrada aqui.

0.2-RC1

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • Anotação @Criteria possui uma nova propriedade, order-by
  • Suporte a agrupamentos específicos de widgets para regras de habilitação e visibilidade, através de enabledWidgetGroup e visibleWidgetGroup
  • Suporte a definição do texto para o elemento vazio de comboboxes e listas
  • @DataProvider pode definir o nome do widget diretamente através da propriedade widgetName. Esta propriedade também permite usar @DataProvider para widgets somente leitura.
  • @DataProvider também pode atualizar o(s) índice(s) do(s) elemento(s) selecionado(s) através da utilização de indexField.
  • @DataProvider também funciona em conjunto com a anotação @ClearOn para limpar o conteúdo do widget correspondente.
  • Suporte a utilização de listas com @DataProvider
  • Pode-se invocar ações de dentro do form com as novas classes ActionInvoker e BaseForm
  • É possível definir formatadores específicos para determinados campos

Os seguintes bugs foram corrigidos nessa versão:

  • A validação deve ocorrer sobre os valores mostrados e não sobre os valores contidos no form (Issue # 134)
  • Dados inválidos para propriedades de tipos primitivos causam uma NullPointerException (Issue # 135)
  • Erros no comportamento de combobox com blank=true afetam o elemento selecionado ou causam uma exceção (Issue # 138)
  • A target genesis.app:run do empty-project não funciona (Issue # 156)
  • Utilização de checkbox sem um grupo faz com que o texto do mesmo seja atualizado (e não o seu estado). Caso o usuário clique no checkbox, uma NullPointerException ocorre (Issue # 165)

Uma lista completa das alterações ocorridas entre o 0.2-beta2 e 0.2-RC1 pode ser obtida aqui.

A documentação completa para esta versão pode ser encontrada aqui.

0.2-beta2

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • Nova classe auxiliar para facilitar o processo de inicialização da aplicação, StartupHelper
  • Suporte a um novo atributo em @DataProvider, callOnInit
  • Suporte a definição de grupos de widgets para uso com as anotações @EnabledWhen / @VisibleWhen
  • Novas classes auxiliares para paginação e pesquisas dinâmicas, BaseSearchForm e BaseCriteriaSearchForm
  • Maior simplicidade no uso do binding, com invokeFormAction(..), refreshView(..) e refreshViewFromForm(Object)
  • Execução em EJB com chamada local
  • Diversos novos parâmetros de configuração e melhorias nos aspectos

Os seguintes bugs foram corrigidos nessa versão:

  • O estado do form não é resetado quando ocorre uma exceção (Issue # 55)
  • CommandResolver.isRemotable(Method) não é sempre invocado (Issue # 74)
  • @DataProvider não funciona corretamente com combos (Issue # 81)
  • @ClearOn não funciona corretamente com @Condition (Issue # 88)
  • Os testes unitários não podem ser executados porque a task <webdoclet> não está definida (Issue # 91)
  • Pode ocorrer um ClassCastException quando se utiliza @DataProvider (Issue # 93)
  • @EnabledWhen/@VisibleWhen não funcionam com propriedades não ligadas (Issue # 110)
  • @Action e @DataProvider em métodos herdados são ignoradas (Issue # 115)
  • Bug do JDK 1.4.2 em PropertyDescriptor.getReadMethod() afeta a configuração da instância de FieldMetadata (Issue # 116)
  • O parsing de anotações para ThinletMetadata ignora métodos herdados (Issue # 122)

Uma lista completa das alterações ocorridas entre o 0.2-beta1 e 0.2-beta2 pode ser obtida aqui.

A documentação completa para esta versão pode ser encontrada aqui.
