Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • O binding de botões de rádio e caixas de seleção foi revisto e documentado

Os seguintes bugs foram corrigidos nessa versão:

  • A validação possui falhas críticas para o Swing e o SWT (Issue # 399).

  • Uma StackOverflow é lançada se indexField for usado com um @DataProvider ligado a um JTable (Issue # 426).

Uma lista completa das alterações ocorridas entre a 3.0-EA5 e a 3.0-RC1 pode ser obtida aqui.

3.0-EA5

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • Foram melhoradas as mensagens de erro para anotações incorretas, propriedades inexistentes e arquivos não disponíveis.
  • O exemplo UserAdmin teve melhorias funcionais para quando o grupo está associado com usuários e quando o SWT não está configrado corretamente.

Os seguintes bugs foram corrigidos nessa versão:

  • all.with.webstart falha se o diretório do SWT não existir (Issue # 376).
  • webstart-jnlp-local falha se jdbc.driver for sobrescrita (Issue # 387).
  • NullPointerException aleatória é lançada ao ler as anotações (Issue # 389).
  • Instâncias de JMenu/JMenuItem não podem ser localizadas por LookupStrategy (Issue # 394).
  • A validação falha para o campo state com Swing no UserAdmin (Issue # 401).

Uma lista completa das alterações ocorridas entre a 3.0-EA4 e a 3.0-EA5 pode ser obtida aqui.

A documentação completa para esta versão pode ser encontrada aqui.

3.0-EA1

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • Suporte genérico a linguagens de script: expandindo o suporte JXPath, o framework suporta diretamente BeanShell, JavaScript e EL, e ainda provê um mecanismo para suporte de outras linguagens
  • Upgrade para o AspectWerkz 2.0. Os aspectos do genesis sofreram pequenas alterações do ponto de vista do usuário e o arquivo aop.xml precisa ser atualizado
