Extensões do XDoclet

O XDoclet é um framework que possibilita a geração de código através de tags Javadoc especiais contidas no código fonte. O desenvolvimento do genesis iniciou-se quando a versão mais recente do XDoclet era a 1.2.0 e diversos recursos ainda não estavam presentes nele, principalmente referente à integração com o Hibernate.

Para contornar estas limitações, durante um bom tempo de sua história a distribuição do genesis conteve alguns módulos modificados do XDoclet. Com a evolução do próprio XDoclet, alguns recursos foram incorporados, mas a forma com que as extensões presentes no genesis eram distribuídas impediam que se usasse qualquer recurso dos módulos sobrepostos - os módulos de validação e do Hibernate. A partir da versão 2.2 do genesis, essa distribuição foi modificada de modo a permitir mais facilmente o uso tanto das funcionalidades originais como as das modificadas.

A versão atual das extensões contém todas as funcionalidades presentes na versão 1.2.3 original do XDoclet, além das modificações explicadas abaixo.

Módulo genesis

O framework Commons Validator nasceu como parte do Struts e, por isso mesmo, foi primeiro suportado pelo XDoclet apenas para funcionar junto com este framework web. Isto explica por que as tags que geram o xml para o Commons Validator possuem originalmente o prefixo @struts. e dependem explicitamente de classes do Struts.

O módulo de extensões do XDoclet chamado genesis suporta tags com a mesma semântica das tags @struts.validator*, exceto que o nome das tags começa com @genesis.validator e funcionam com POJOs ao invés de requerer que a classe que possui as anotações estenda alguma classe do Struts. Um exemplo do uso das tags pode ser encontrado na documentação do suporte à validação.

Módulo hibernate

O módulo hibernate contém algumas funcionalidades não presentes no módulo original fornecido com o XDoclet. As principais delas são suporte a geração do elemento meta através da tag @hibernate.meta, definição de relacionamentos many-to-one para tabelas com chaves compostas, merge de queries em arquivos externos e também do mapeamento de componentes. Para maiores informações, favor consultar o issue submetido ao projeto do XDoclet que contém as melhorias feitas pela equipe do genesis.
