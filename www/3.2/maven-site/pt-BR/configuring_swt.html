Configurando as bibliotecas do SWT

O SWT (Standard Widget Toolkit) possui diferentes distribuições para diversas plataformas. Desta forma, aqueles que usam Windows devem possuir a distribuição específica para Windows, assim como usuários Linux, Solaris ou Mac OS devem utilizar a distribuição de sua plataforma. Abaixo encontram-se instruções de como deve ser configurada uma ou mais distribuições para uso num projeto junto com o genesis.

Nenhuma distribuição do SWT acompanha a distribuição do genesis, logo, para criar um projeto SWT ou rodar a parte SWT do exemplo useradmin é necessário baixar a distribuição SWT correspondente para sua plataforma. Os arquivos podem ser encontrados no site oficial do SWT.

Os arquivos da distribuição desejada devem ser descompactados na pasta <genesis.home>/genesis/lib/swt e organizados hierarquicamente por sistema operacional e arquitetura do processador. Por exemplo, imagine um projeto onde será utilizado SWT em máquinas Windows (processador x86 - 32 bits), Linux (usando GTK e processadores x86 - 32 e 64 bits), Solaris (processador SPARC) e Mac OS (processador PPC). A estrutura criada em <genesis.home>/genesis/lib/swt será:

Diretório <genesis.home>/genesis/lib/swt
  
Windows
  + x86
    - swt.jar
    - swt1.dll
    - swt2.dll
    - etc...

Linux
  + x86_64
    - swt.jar
    - swt1.so
    - swt2.so
    - etc...  
  - swt.jar
  - swt1.so
  - swt2.so
  - etc...

Solaris
  + sparc
    - swt.jar
    - swt1.so
    - swt2.so
    - etc...
  
Mac
  - swt.jar
  - swt1.jnilib
  - swt2.jnilib
  - etc...

O nome da pasta correspondente ao sistema operacional corresponde ao nome da propriedade de sistema da JVM os.name. É possível utilizar apenas a parte inicial do valor de os.name.

O nome da pasta correspondente a arquitetura do processador corresponde ao nome da propriedade de sistema da JVM os.arch.

Para conhecer os valores das propriedades os.name e os.arch para determinado ambiente, compile e rode o programa abaixo:

public class OsAndArch {
  public static void main(String[] args) {
     System.out.println("Sistema Operacional: " + System.getProperty("os.name"));
     System.out.println("Arquitetura Processador: " + System.getProperty("os.arch"));
  }  
}

Para o Windows XP, a seguinte saída é visualizada:

Sistema Operacional: Windows XP
Arquitetura Processador: x86

Neste caso, os jars e dlls do SWT devem ser colocados no diretório Windows/x86.
