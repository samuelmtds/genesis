Definindo os jars necessários para sua aplicação

É possível utilizar somente o binding do genesis sem usar sua estrutura de projeto que vem com a distribuição do framework. Contudo, os jars necessários variam de acordo com as funcionalidades e a versão mínima do JRE que será utilizada. A tabela abaixo descreve os requisitos para cada cenário:

Funcionalidade Java 1.4 Java 5 Java 6 Observação
Binding backport175-<versão>.jar

commons-beanutils-<versão>.jar

commons-logging-<versão>.jar

genesis-client-<versão>.jar

genesis-shared-annotated-<versão>.jar

reusable-components-<versão>.jar
Suporte a anotações genesis-annotation-jdk14-<versão>.jar genesis-annotation-jdk5-<versão>.jar
Binding Swing genesis-client-swing-<versão>.jar
Binding SWT genesis-client-swt-<versão>.jar
Binding Thinlet genesis-client-thinlet-<versão>.jar
JavaScript como linguagem de script bsf-<versão>.jar

js-<versão>.jar 		Nenhum jar é necessário Essa é a linguagem de script padrão
BeanShell como linguagem de script bsf-<versão>.jar

bsh-<versão>.jar
EL como linguagem de script bsf-<versão>.jar

jakarta-taglibs-standard-<versão>.jar

jsp-api.jar
