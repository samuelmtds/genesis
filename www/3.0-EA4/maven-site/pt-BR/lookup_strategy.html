Estratégias de pesquisa de componentes gráficos

Um dos conceitos básicos do binding implementado pelo genesis é determinar qual (e se existe um) componente representa uma propriedade, @Action ou @DataProvider. Dessa forma, a interface gráfica possuirá componentes identificados por nome que deverão ser acessíveis às classes de binding de alguma forma. A maneira utilizada pelo binding para procurar tais componentes chamamos de estratégia de pesquisa de componentes gráficos.

O genesis define a interface net.java.dev.genesis.ui.binding.LookupStrategy que possui os seguintes métodos:

  • register(String name, Object widget): Associa um widget a um nome.
  • lookup(Object root, String name): Faz a busca por um widget registrado com o nome especificado dentro do escopo do widget root passado como parâmetro.
  • getName(Object widget): Dado um widget, retorna o seu nome de registro.

Existe uma implementação básica e genérica que utiliza um java.util.Map como sua estrutura de dados. Esta implementação é definida na classe net.java.dev.genesis.ui.binding.MapLookupStrategy e pode ser utilizada independente da tecnologia gráfica utilizada.

Existem estratégias de pesquisa específicas para cada tecnologia gráfica utilizada. No momento, o genesis suporta três tecnologias gráficas: Thinlet, Swing e SWT. Como o Thinlet já possui um suporte nativo a busca de componentes pelo nome, o genesis define implementações específicas para Swing e SWT. A seguir, veremos as implementações específicas para cada uma dessas tecnologias.

Swing

Existem três implementações de LookupStrategy providas pelo genesis para Swing. Cada uma dessas implementações utiliza o nome do componente (java.awt.Component.getName()) para a busca do mesmo:

  • MapComponentLookupStrategy: Utiliza o mesmo mecanismo de MapLookupStrategy. A única diferença está no método getName(Object widget) que, ao invés de retornar null caso o componente não esteja registrado, retorna o nome do componente (component.getName()).
  • BreadthFirstComponentLookupStrategy: Não é necessário utilizar o método register para registrar componentes, pois é utilizado o próprio nome do componente para a sua busca (Component.getName()). A implementação de seu método lookup, recebe o componente raiz e inicia uma busca em largura em todos os componentes que o componente raiz possui.
  • DepthFirstComponentLookupStrategy: Semelhante ao BreadthFirstComponentLookupStrategy, exceto pelo algoritmo de busca, que nesse caso é o algoritmo de busca em profundidade.

A implementação padrão utilizada pelo genesis é BreadthFirstComponentLookupStrategy. É possível mudar a implementação utilizada pelo binding sobrescrevendo o método createLookupStrategy() da classe net.java.dev.genesis.ui.swing.SwingBinder ou definindo a implementação desejada no construtor de SwingBinder.

SWT

Assim como em Swing, existem três implementações de LookupStrategy providas pelo genesis para SWT. Cada uma dessas implementações utiliza o nome do widget (org.eclipse.swt.widgets.Widget.getData()) para a busca do mesmo:

  • MapWidgetLookupStrategy: Utiliza o mesmo mecanismo de MapLookupStrategy. A única diferença está no método getName(Object widget) que, ao invés de retornar null caso o widget não esteja registrado, ele retorna o nome do widget (Widget.getData()).
  • BreadthFirstWidgetLookupStrategy: Não é necessário utilizar o método register para registrar componentes, pois é utilizado o próprio nome do componente para a sua busca (Widget.getData()). A implementação de seu método lookup, recebe o componente raiz e inicia uma busca em largura em todos os componentes que o componente raiz possui.
  • DepthFirstWidgetLookupStrategy: Semelhante ao BreadthFirstWidgetLookupStrategy, exceto pelo algoritmo de busca, que nesse caso é o algoritmo de busca em profundidade.

Observe que diferentemente do Swing, o SWT não possui um método para definir o nome para seus componentes. Para definir os nomes dos componentes SWT, utilize Widget.setData("nomeDesejado").

A implementação padrão utilizada pelo genesis é BreadthFirstWidgetLookupStrategy. É possível mudar a implementação utilizada pelo binding sobrescrevendo o método createLookupStrategy() da classe net.java.dev.genesis.ui.swt.SWTBinder ou definindo a implementação desejada no construtor de SWTBinder.
