0.2-beta2

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Algumas das novas funcionalidades e mudanças mais notáveis são:

  • Nova classe auxiliar para facilitar o processo de inicialização da aplicação, StartupHelper
  • Suporte a um novo atributo em @DataProvider, callOnInit
  • Suporte a definição de grupos de widgets para uso com as anotações @EnabledWhen / @VisibleWhen
  • Novas classes auxiliares para paginação e pesquisas dinâmicas, BaseSearchForm e BaseCriteriaSearchForm
  • Maior simplicidade no uso do binding, com invokeFormAction(..), refreshView(..) e refreshViewFromForm(Object)
  • Execução em EJB com chamada local
  • Diversos novos parâmetros de configuração e melhorias nos aspectos

Os seguintes bugs foram corrigidos nessa versão:

  • O estado do form não é resetado quando ocorre uma exceção (Issue # 55)
  • CommandResolver.isRemotable(Method) não é sempre invocado (Issue # 74)
  • @DataProvider não funciona corretamente com combos (Issue # 81)
  • @ClearOn não funciona corretamente com @Condition (Issue # 88)
  • Os testes unitários não podem ser executados porque a task <webdoclet> não está definida (Issue # 91)
  • Pode ocorrer um ClassCastException quando se utiliza @DataProvider (Issue # 93)
  • @EnabledWhen/@VisibleWhen não funcionam com propriedades não ligadas (Issue # 110)
  • @Action e @DataProvider em métodos herdados são ignoradas (Issue # 115)
  • Bug do JDK 1.4.2 em PropertyDescriptor.getReadMethod() afeta a configuração da instância de FieldMetadata (Issue # 116)
  • O parsing de anotações para ThinletMetadata ignora métodos herdados (Issue # 122)

Uma lista completa das alterações ocorridas entre o 0.2-beta1 e 0.2-beta2 pode ser obtida aqui.

0.2-beta1

Downloads

Todos os arquivos de distribuição para esta versão podem ser obtidos aqui.

Funcionalidades e mudanças

Diversas mudanças e funcionalidades novas fazem parte desta versão, entre outras podendo-se destacar:

  • Métodos remotos e transacionais são definidos através de anotações e não mais com marker interfaces + convenção de nomes para métodos
  • Implementação de @Criteria
  • Modo de execução local
  • Modelo de interface gráfica totalmente reformulado e muito mais expressivo e poderoso
